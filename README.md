# drf_oroject
Учебный проект API для системы управления курсами и уроками, созданный с помощью Django REST Framework.
# Описание
Этот проект реализует API для LMS (Learning Management System), позволяя работать с сущностями:

- **Course** — курсы с названием, превью, описанием.
- **Lesson** — уроки, связанные с курсами, с названием, описанием, превью и видео ссылкой.

Реализованы CRUD-операции (создание, чтение, обновление, удаление) для курсов и уроков.
## Особенности
- Python 3.12
- Django 5.2
- Django REST Framework
- PostgreSQL
- Использование ViewSet и Generic Views DRF

## Установка и запуск
Клонировать репозиторий:
~~~
git clone https://github.com/natali83911/drf_project.git
cd drf_project
~~~

Создать и активировать виртуальное окружение:
~~~
python -m venv venv
source venv/bin/activate # Linux/macOS
venv\Scripts\activate # Windows
~~~

Установить зависимости:
~~~
pip install -r requirements.txt
~~~

Применить миграции и создать суперпользователя:
~~~
python manage.py migrate
python manage.py createsuperuser
~~~

Запустить сервер разработки:
~~~
python manage.py runserver
~~~

## Использование API

### Курсы

- `GET /lms/courses/` - получить список курсов
- `POST /lms/courses/` - создать новый курс
- `GET /lms/courses/<id>/` - получить информацию о курсе
- `PUT /lms/courses/<id>/` - обновить курс
- `DELETE /lms/courses/<id>/` - удалить курс

### Уроки

- `GET /lms/lessons/` - получить список уроков
- `POST /lms/lessons/` - создать новый урок
- `GET /lms/lessons/<id>/` - получить информацию об уроке
- `PUT /lms/lessons/<id>/` - обновить урок
- `DELETE /lms/lessons/<id>/` - удалить урок

## Контакты

По вопросам и предложениям обращайтесь к автору проекта.


Структура
~~~
drf_project/                  # Корневая директория проекта
│
├── config/                   # Основная конфигурация проекта (settings, urls, wsgi/asgi)
│   ├── __init__.py
│   ├── settings.py           # Основные настройки Django
│   ├── urls.py               # Корневые маршруты URL проекта
│   ├── wsgi.py
│   └── asgi.py
│
├── lms/                      # Приложение LMS: курсы и уроки
│   ├── migrations/           # Миграции базы данных
│   ├── __init__.py
│   ├── admin.py              # Регистрация моделей в админке
│   ├── apps.py
│   ├── models.py             # Определение моделей Course и Lesson
│   ├── serializers.py        # Сериализаторы для моделей LMS
│   ├── views.py              # ViewSet и Generic Views для курсов и уроков
│   ├── urls.py               # URL-маршруты приложения LMS (курсы, уроки)
│   └── tests.py              # Тесты приложения
│
├── users/                    # Приложение пользователей (кастомная модель User)
│   ├── migrations/
│   ├── __init__.py
│   ├── admin.py
│   ├── apps.py
│   ├── models.py             # Пользовательская модель User (email, телефон, город, аватар)
│   ├── serializers.py
│   ├── views.py
│   └── urls.py
│
├── manage.py                 # Утилита управления проектом
├── requirements.txt          # Зависимости проекта
├── .env                      # Переменные окружения (секреты, настройки БД)
└── README.md                 # Описание проекта (этот файл)

~~~
Ключевые моменты:
- config/ — настройки и главный файл URL, в котором подключаются все приложения.
- lms/ — логика учебных курсов и уроков: модели, сериализаторы, представления и url.
- users/ — кастомная модель пользователя с дополнительными полями.
- Каждый модуль обладает собственной структурой для удобства поддержки.
- Миграции хранятся в папках migrations/ каждого приложения.
- Все маршруты API подключаются через роутер или ручные пути в urls.py каждого приложения.-
- В корневом config/urls.py обычно подключают URLы всех приложений.
